Question Answer Generation via Tables (Read them carefully)

In this task, we ask you to create temporal (time-related) questions based on a table and provide answers to these questions you create. In particular,

Given a table, you should write a question and answer based on

  1. time-related facts appearing in the table (using only the table),
  2. what you feel most people know about the world (temporal common sense).

To create meaningful questions, there are a few points to keep in mind:

  • Don't write questions that don't have temporal aspects.
    • Your questions should be self-contained. That is, even if the table is removed question still make sense.
    • Don't use pronouns "He", "She", "They" etc in writing questions
    • Please write grammatically correct questions and also provide answers.
    • For each question, the answer should be unambiguous and unique.
  • Write a question that would require some inference to answer instead of one explicitly mentioned in the tables.
    • Please avoid writing trivial questions that require only looking up from the table.
    • Your question should include an event, or ask about an event, and should not be a yes/no question.
    • Disregard the picture on the table top. Don't ask any question about the picture or caption.
  • If you annotate multiple HITs try to write diverse questions across HITs.
    • Try to avoid writing questions with similar patterns across HITs.
    • Across HITs write questions which have diverse temporal reasoning.
    • Don't merely rephrase questions from earlier HITs with different table terms.
  • When drafting multiple questions with a HIT, be sure that each one uses a different table sections and has diverse logical reasoning.
    • Use diverse parts of the table rows across questions. Avoid concentrating on only one part of the table for all questions.
    • Incorporate different logical aspects i.e. required reasoning across questions.
    • Write questions with varied starting tokens i.e., "What", "When", "Whose", "How", "Where", "Who", "How many", etc. across questions, if possible.
  • When writing a question, use information across different parts of the table.
    • Ideally, use multiple rows and columns to create a challenging question.
    • Concise and simple "answer" are preferred over verbose ones.
    • Answers should not be sentences long, but rather phrases conveying the exact answer.
  • Your question must be related to the table and be answerable with common sense.
    • Don't use names of people or places not mentioned in the table.
    • Also, don't refer to outside sources of information.
    • Incorporate more variations and complexity to the questions. In other words, we want you to use "instinct" or "common sense".
  • Encourage to use of temporal terms to represent time categories such as the following.
    • Category Terms
    before before, previous to, prior to, preceding, followed by, until
    after after, following, since, once, now that, soon after
    during during, while, when at the time, at the same time, meanwhile
    past earlier, previously, formerly, in the past, yesterday, last time
    future consequently, subsequently, in turn, henceforth, later, then
    beginning initially, originally, at the beginning, to begin, starting with, to start with
    ending finally, in the end, at last, lastly
